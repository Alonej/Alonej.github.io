hexo搭建及部署

一、Hexo本地建站

1.hexo配置

1.1 安装

安装Hexo(https://hexo.io/zh-cn/docs/) 前提是电脑要有Nodes.js 和 Git

具备以上条件后安装hexo只需在命令行（cmd、Git bash）执行

1.2 建站

新建一个’your_name.github.io’的文件夹，命令行cd到此文件夹，执行 npm install。

如果速度太慢的话把默认VPM镜像源换为淘宝的镜像，执行

完成后可执行

命令(cmd路径要在刚建的文件夹下)，访问http://localhost:4000/查看我们的默认博客。

_config.yml为配置文件，可用记事本打开，网站的配置信息

1.3 主题设置

在Hexo官网主题里找到自己喜欢的，然后找到它github的主页，这里总结了些许主题的项目主页。

下载主题可以通过执行 git clone 主题项目网址 themes/name

下载渲染器:

下载渲染器可能会报错，先换下镜像源试下。下好的主题放到themes文件夹里。

更换主题只要把_config.yml文件里theme: landscape 改为你下载的主题文件名，注意冒号后面要有空格

主题更换完后，执行 hexo clean 、hexo g、hexo s，再看下主题变没变，如果是一对字符，可能是主题有问题，换个主题试下。

2.1创建一个仓库

2.3 SSH

2.31 生成SSH

生成秘钥，注意C是大写，根据步骤按回车，passphrase可以不填。

将生成两个文件，id_rsa为私钥，id_rsa.pub为公钥

2.32 绑定SSH

在Git bash执行

用记事本打开公钥id_rsa.pub，复制所有内容，点击Settings>SSH keys,将内容粘贴到这里，title为“yourname.github.io”

添加成功后再终端测试下，执行ssh -T git@github.com

You’ve successfully authenticated, but GitHub does not provide shell access.就说明成功了，若出现Are you sure you want to continue connecting (yes/no/[fingerprint])? YES就好了
